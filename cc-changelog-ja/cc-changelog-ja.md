# Claude Code 変更履歴

## 2.0.69

- 軽微なバグ修正

## 2.0.68

- 中国語、日本語、韓国語などの言語における IME (Input Method Editor) サポートを修正し、コンポジションウィンドウをカーソル位置に正しく配置するように
- 許可されていない MCP ツールがモデルに見えてしまうバグを修正
- サブエージェントの動作中にステアリングメッセージが失われる可能性がある問題を修正
- Option+Arrow による単語移動で、CJK (中国語、日本語、韓国語) のテキスト列全体が単語境界ではなく 1 つの単語として扱われる問題を修正
- プランモード終了時の UX を改善: 空のプランやプランがない状態で終了する際に、エラーをスローするのではなく、簡略化された yes/no ダイアログを表示するように
- エンタープライズ管理設定のサポートを追加。この機能を有効にするには、Anthropic アカウントチームにお問い合わせください。

## 2.0.67

- Claude がワークフローを高速化するためのプロンプトを提案するように（Tab で承認、Enter で送信）
- Opus 4.5 で思考モードがデフォルトで有効に
- 思考モードの設定が /config に移動
- `/permissions` コマンドに検索機能を追加（`/` ショートカットでツール名によるフィルタリングが可能）
- `/doctor` で自動更新が無効化されている理由を表示するように
- 別のインスタンスが既に最新バージョンである状態で `claude update` を実行した際に誤って「Another process is currently updating Claude」エラーが表示される問題を修正
- 非対話モード（`-p` フラグまたはパイプ入力）実行時に `.mcp.json` の MCP サーバーが保留状態のままになる問題を修正
- `/permissions` で権限ルールを削除した後にスクロール位置がリセットされる問題を修正
- キリル文字、ギリシャ文字、アラビア文字、ヘブライ文字、タイ文字、中国語などの非ラテン文字で、単語削除（opt+delete）や単語移動（opt+arrow）が正しく機能しない問題を修正
- `claude install --force` が古いロックファイルをバイパスしない問題を修正
- Markdown の取り消し線の干渉により、CLAUDE.md 内の連続する @~/ ファイル参照が誤って解析される問題を修正
- Windows: ログディレクトリパスに含まれるコロンが原因でプラグイン MCP サーバーが失敗する問題を修正

## 2.0.65

- プロンプト入力中に alt+p (Linux, Windows) または option+p (macOS) でモデルを切り替える機能を追加
- ステータスライン入力にコンテキストウィンドウ情報を追加
- カスタム `@` ファイル検索コマンド用の `fileSuggestion` 設定を追加
- 自動シェル検出を上書きするための `CLAUDE_CODE_SHELL` 環境変数を追加（ログインシェルと実際の作業シェルが異なる場合に便利）
- Escape キーでクエリを中止した際にプロンプトが履歴に保存されない問題を修正
- ファイル拡張子の代わりにバイトデータから形式を識別するように Read ツールの画像処理を修正

## 2.0.64

- 自動コンパクト化が瞬時に行われるように
- エージェントと bash コマンドが非同期で実行され、メインエージェントをウェイクアップするメッセージを送信可能に
- `/stats` で、お気に入りのモデル、使用量グラフ、使用ストリークなど、興味深い CC 統計情報をユーザーに提供
- 名前の付いたセッションのサポートを追加：`/rename` でセッションに名前を付け、REPL で `/resume <name>` またはターミナルから `claude --resume <name>` で再開可能に
- `.claude/rules/` のサポートを追加。詳細は https://code.claude.com/docs/en/memory を参照
- 画像リサイズ時に画像の寸法メタデータを追加し、大きな画像の正確な座標マッピングを可能に
- ネイティブインストーラー使用時に .env の自動読み込みが機能しない問題を修正
- `--continue` または `--resume` フラグ使用時に `--system-prompt` が無視される問題を修正
- 分岐したセッションのグループ化、プレビュー (P) と名前変更 (R) のキーボードショートカットを追加し、`/resume` 画面を改善
- VSCode: コードブロックと bash ツール入力にクリップボードへのコピーボタンを追加
- VSCode: エミュレーションを介して x64 バイナリにフォールバックすることで、Windows ARM64 で拡張機能が機能しない問題を修正
- Bedrock: トークンカウントの効率を改善
- Bedrock: `aws login` AWS Management Console の認証情報のサポートを追加
- 新しい統合された TaskOutputTool を採用し、AgentOutputTool と BashOutputTool を廃止
